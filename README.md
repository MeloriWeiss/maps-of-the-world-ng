# Maps-of-the-World

## О продукте

Проект создан с помощью Angular CLI 19.2.14 + Nx.

---

## Структура проекта

Монорепозиторий (nx workspaces):

| Пакет           | Назначение                                                                |
| --------------- | ------------------------------------------------------------------------- |
| **libs/shared** | Общие типы, DTO, конфигурации, утилиты. Используется libs/web и libs/api. |
| **libs/api**    | библиотеки для бэкенда                                                    |
| **libs/web**    | библиотеки для фронтенда                                                  |
| **apps/api**    | HTTP API на Nest.js                                                       |
| **apps/web**    | SPA на Angular                                                            |

## Запуск

### Подготовка

Важно! Перед установкой пакетов убедитесь, что ваша версия node.js соответствует той, которая указана в `.nvmrc`.

Для установки зависимостей, используйте:

```bash
yarn install
```

Далее создайте `.env` в корне проекта на основе `.env.example` и заполните тестовыми данными.

### Разработка

#### 1. Фронтенд

Для запуска фронтенда в режиме разработки:

```bash
yarn start:web
```

#### 2. Бэкенд

Для запуска бэкенда в режиме разработки:

```bash
yarn start:api
```

#### 3. Окружение

Для запуска приложений, необходимых для работы бэкенда в режиме разработки (требуется `Docker`):

```bash
yarn start:api-deps:docker
```

Для подготовки базы данных к использованию используйте следующие команды в соответствующем порядке:

```bash
yarn db:main:push
```

```bash
yarn db:main:generate
```

```bash
yarn db:main:seed
```

### Сборка

#### 1. Фронтенд

Для сборки фронтенда:

```bash
yarn build:web:prod
```

#### 2. Бэкенд

Для сборки серверной части, создайте `.env.prod` в `docker/` и заполните тестовыми данными на основе `.env.prod.example`.

Для сборки бэкенда и приложений, необходимых для его работы на сервере, в одном `docker`-образе:

```bash
yarn build:api:prod:docker
```

База данных поднимается на сервере отдельно, без Docker

## Команды

### Из корня репозитория

| Команда                      | Описание                                                                                     |
| ---------------------------- | -------------------------------------------------------------------------------------------- |
| `yarn start:api`             | Запуск только бэкенда                                                                        |
| `yarn start:web`             | Запуск только фронтенда                                                                      |
| `yarn start:api-deps:docker` | Запуск приложений, необходимых для работы бэкенда, в Docker                                  |
| `yarn build:api:prod`        | Сборка бэкенда `для сервера`                                                                 |
| `yarn build:web:prod`        | Сборка фронтенда `для сервера`                                                               |
| `yarn build:api:prod:docker` | Сборка Docker-конейнеров с приложениями, необоходимыми для работы бэкенда, `для сервера`     |
| `yarn db:main:migrate`       | Применить миграцию к main базе данных                                                        |
| `yarn db:main:generate`      | Сгенерировать типы на основе схемы для main базы данных                                      |
| `yarn db:main:seed`          | Выполнить сидинг базы данных (заполнить тестовыми данными)                                   |
| `yarn db:main:push`          | Применить все изменения к базе данных (например, чтобы создать таблицы в пустой базе данных) |
| `yarn lint`                  | Запустить линтер по всему монорепозиторию                                                    |
| `yarn test`                  | Запустить тесты по всему монорепозиторию                                                     |
| `yarn docs:gen`              | Сгенерировать документацию ко всему монорепозиторию                                          |
| `yarn docs:watch`            | Запустить сервер для просмотра документации                                                  |

## Качество кода

- **ESLint + Prettier** — линтинг и форматирование; кастомные правила в `eslint-rules` (см. [`tools/eslint-rules/README.md`](tools/eslint-rules/README.md)).
- **Husky + lint-staged** — перед коммитом: форматирование и линтинг изменённых файлов.
- **Commitlint** — формат коммитов по [Conventional Commits](https://www.conventionalcommits.org/) (в т.ч. русский текст). Подробнее: [COMMIT_CONVENTION.md](COMMIT_CONVENTION.md).

Примеры коммитов:

```bash
feat(web): добавить компонент UserCard
fix(api): исправить обработку ошибок
docs: обновить README
```
